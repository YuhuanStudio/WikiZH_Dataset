# 貢獻指南

感謝你對 WikiZH_Dataset 的興趣！我們歡迎所有形式的貢獻。

## 如何貢獻

### 報告錯誤

如果你發現了錯誤，請透過 [GitHub Issues](../../issues) 回報。在提交 Issue 之前，請先搜尋現有的 Issue 以避免重複。

### 建議新功能

歡迎提出新功能的建議。請詳細描述你想要的功能以及使用場景。

### 提交程式碼

1. **Fork 這個專案**
2. **建立功能分支**：`git checkout -b feature/your-feature-name`
3. **進行變更**：
   - 確保程式碼符合現有風格
   - 新增或更新相關測試
   - 更新文件
4. **提交變更**：`git commit -m 'Add some feature'`
5. **推送到你的 Fork**：`git push origin feature/your-feature-name`
6. **提交 Pull Request**

## 程式碼規範

### Python

- 遵循 PEP 8 風格指南
- 使用有意義的變數和函數名稱
- 新增適當的文件字串（docstrings）
- 保持函數簡短並專注於單一責任

### 文件

- 使用 Markdown 格式
- 保持描述簡潔明確
- 包含使用範例

## 開發環境設定

```bash
# 複製專案
git clone https://github.com/YuhuanStudio/WikiZH_Dataset.git
cd WikiZH_Dataset

# 安裝相依套件
python -m pip install -r requirements.txt
```

## 提交訊息格式

使用清晰的提交訊息格式：

- `feat: 新增新功能`
- `fix: 修復錯誤`
- `docs: 更新文件`
- `style: 程式碼格式調整`
- `refactor: 程式碼重構`
- `test: 新增或更新測試`
- `chore: 其他變更`

## 程式碼審查

所有的 Pull Request 都需要經過審查。審查者可能會要求變更，請保持耐心並積極回應。

## 行為準則

- 尊重所有貢獻者
- 建設性地提供回饋
- 保持對話專業和友善

## 授權

提交你的貢獻代表你同意你的貢獻將使用與本專案相同的授權（Apache 2.0）。

有任何問題嗎？歡迎在 Issue 中提問！
